XML y Bases de Datos

1. Vamos a utilizar la base de datos catalog (www.rvg.ua.es:1521:j2eebd), que contiene dos tablas: categories, y category_products, con los siguientes campos:

  • Tabla categories: categid (código de la categoría), categname (nombre de la categoría), categdesc (descripción de la categoría)
  • Tabla category_products: productid (identificador del producto), categid (código de la categoría), prodname (nombre del producto), procdesc (descripción del producto).

Utilizando la utilidad XSU, crear las clases correspondientes para realizar las siguientes operaciones en la base de datos anterior:

  • Dado un nombre de categoría, que se pasa como parámetro de entrada, obtener un fichero XML con la lista de productos de dicha categoría.
  • Dado un fichero XML de entrada, con cinco productos, insertarlos en la base de datos, de forma que si su identificador de categoría es nuevo, se añada dicho identificador en la tabla de categorías.
  • Dado un nombre de categoría, que se pasa como parámetro de entrada, actualizar la descripción de los productos de la base de datos con esa categoría, añadiendo el texto: "consumir antes de fin de mes".


2. Utilizando la base de datos anterior, pero en la que hay los siguientes campos adicional es:

  • en la tabla categories: campo prodschema (documento XML con el formato de los datos de la categoría)
  • en la tabla category_products: campo prodxml (documento XML con los datos del producto)

Implementar las clases correspondientes para realizar las siguientes operaciones en dicha base de datos:

  • Dado un nombre de categoría, que se pasa como parámetro de entrada, obtener un fichero con un listado de los campos prodxml de todos productos de dicha categoría.
  • Dado un fichero XML de entrada con los datos correspondientes al campo prodxml de un producto, un nombre de producto, una descripción y un identificador de categoría y de producto, añadir dichos datos a la tabla de categorías de productos. Si el identificador de categoría ya existe, añadirlo en la tabla de categorías.
  • Dado un porcentaje de descuento, que se pasa como parámetro de entrada, actualizar el precio de los productos de la base de datos cuyo precio después del descuento sea superior o igual a 7.
